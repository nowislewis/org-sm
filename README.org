#+title: org-sm
#+author: Lewis Liu

Minimal incremental reading for org-mode, inspired by SuperMemo.

* Card Types

*topic* — repeated reading passage.  No grading; interval grows by A-Factor each review.

| Priority      | A-Factor | Behavior                       |
|---------------+----------+--------------------------------|
| =[#A]=        |      1.2 | slow growth, review frequently |
| =[#B]=        |      1.5 | moderate                       |
| =[#C]= / none |      1.8 | fast growth, fades out quickly |

*cloze* — fill-in-the-blank with ~{{answer}}~ markers, scheduled by [[https://github.com/bohonghuang/fsrs.el][FSRS]].
Answers are hidden as ~[___]~ during review; rated again / hard / good / easy.

* Installation

Requires Emacs 29.1+, org 9.6+, [[https://github.com/bohonghuang/fsrs.el][fsrs]].

#+begin_src elisp
(use-package org-sm
  :hook (org-mode . org-sm-mode)
  :config
  (setq org-sm--files (directory-files-recursively "~/notes/" "\\.org$"))
  (setq org-sm-capture-file "~/notes/inbox.org")
  (setq org-sm-capture-olp '("org-sm topics"))
  (org-sm-setup-capture)
  (global-org-sm-read-mode 1))  ; M-z captures region/clipboard as topic from any buffer
#+end_src

Suggested keybindings:

#+begin_src elisp
(with-eval-after-load 'org-sm
  (define-key org-mode-map (kbd "C-c s m")   #'org-sm-item-mark)
  (define-key org-mode-map (kbd "C-c s x")   #'org-sm-item-extract)
  (define-key org-mode-map (kbd "C-c s r")   #'org-sm-review-start)
  (define-key org-mode-map (kbd "C-c s SPC") #'org-sm-review-confirm)
  (define-key org-mode-map (kbd "C-c s d")   #'org-sm-item-dismiss)
  (define-key org-mode-map (kbd "C-c s q")   #'org-sm-review-abort)
  (define-key org-mode-map (kbd "C-c s l")   #'org-sm-review-list)
  (define-key global-map   (kbd "C-c s c")   #'org-sm-capture-topic))
#+end_src

* Commands

| Command                 | Description                                     |
|-------------------------+-------------------------------------------------|
| ~org-sm-item-mark~      | Mark current heading as topic or cloze          |
| ~org-sm-item-extract~   | Extract selected region as child topic or cloze |
| ~org-sm-item-dismiss~   | Remove current item from SRS                    |
| ~org-sm-review-start~   | Start review session                            |
| ~org-sm-review-confirm~ | Confirm topic read / advance cloze state        |
| ~org-sm-review-abort~   | Abort current review session                    |
| ~org-sm-review-list~    | Browse all SRS items                            |
| ~org-sm-capture-topic~  | Capture region or clipboard as a topic card     |

* Review Flow

*Topic:* ~review-start~ → read → ~review-confirm~ (reschedules, next item).

*Cloze:* ~review-start~ (answers hidden) → ~review-confirm~ (reveals) → ~review-confirm~ (rate).

Intervals are calculated from the scheduled date, so early/late reviews don't distort spacing.

* Capture

~org-sm-capture-topic~ captures a region or clipboard content as a topic card via ~org-capture~.
With a region active, triggers ~M-w~ first (works in any buffer: pdf, nov, etc.) then reads
the clipboard.  Without a region, reads the clipboard directly.

Set ~org-sm-capture-file~ and ~org-sm-capture-olp~, then call ~org-sm-setup-capture~ once.
~global-org-sm-read-mode~ binds ~M-z~ globally to ~org-sm-capture-topic~.

* Extract

Select text, then call ~org-sm-item-extract~:

- *topic* — selected text becomes the child body.
- *cloze* — full parent body copied to child; selection wrapped as ~{{answer}}~.

The selection in the parent is replaced with an ~[[id:...][title]]~ link.

* Properties

| Property         | Type              | Description                     |
|------------------+-------------------+---------------------------------|
| ~SRS_TYPE~       | ~topic~ / ~cloze~ | Card type                       |
| ~SRS_LAST~       | ISO 8601          | Last review timestamp           |
| ~SRS_STATE~      | symbol            | FSRS state (cloze only)         |
| ~SRS_STABILITY~  | float             | FSRS stability (cloze only)     |
| ~SRS_DIFFICULTY~ | float             | FSRS difficulty (cloze only)    |
| ~SRS_STEP~       | int               | FSRS learning step (cloze only) |

~SCHEDULED~ is the due date for both types.

* AI Assistant (org-sm-gptel)

Optional extension powered by [[https://github.com/karthink/gptel][gptel]].

#+begin_src elisp
(with-eval-after-load 'gptel
  (require 'org-sm-gptel)
  (define-key org-mode-map (kbd "C-c s ?") #'org-sm-gptel-explain)
  (define-key org-mode-map (kbd "C-c s i") #'org-sm-gptel-refine)
  (define-key org-mode-map (kbd "C-c s M") #'org-sm-gptel-generate-concept))
#+end_src

** Commands

*** ~org-sm-gptel-explain~ (=C-c s ?=)

Streams an AI explanation into a side window without interrupting the review.
For cloze: explains the concept behind ~{{}}~ and flags interference risks.
For topic: highlights the most card-worthy facts and explains jargon.
Press ~q~ to abort and close.

*** ~org-sm-gptel-refine~ (=C-c s i=)

Refines the current heading body in-place using ~gptel-rewrite~.
Works on any org heading — including org-capture buffers before ~C-c C-c~.

The system prompt is chosen automatically by card type:

- *cloze* → enforces minimum-information principle: one ~{{}}~ per card,
  answer ≤ 5 words, split multi-fact content into separate cards.
- *topic / unset* → cleans prose for incremental reading: strips filler,
  resolves pronoun ambiguity, preserves paragraph structure (not a summary).

The instruction is pre-filled in the minibuffer; press ~RET~ to confirm or
edit it first.  gptel-rewrite overlay keys:

| Key       | Action          |
|-----------+-----------------|
| ~C-c C-a~ | Accept rewrite  |
| ~C-c C-k~ | Reject rewrite  |
| ~C-c C-d~ | Diff view       |
| ~C-c C-e~ | Ediff view      |
| ~C-c C-r~ | Iterate / retry |

Typical workflow after ~org-sm-capture-topic~ or ~org-sm-item-extract~:
1. Land on the new child heading.
2. ~C-c s i~ to refine raw content.
3. ~C-c C-a~ to accept, then continue.

*** ~org-sm-gptel-generate-concept~ (=C-c s M=)

Synthesizes the current subtree (or active region) into a concept-map Topic.
Inserts a new =[#A]= heading immediately after the subtree, scheduled for SRS.

Output structure:
1. *Core logic thread* — causal / chronological chain linking the items.
2. *Confusion analysis* — side-by-side "decisive feature" comparison of easily
   mixed concepts (table format when applicable).
3. *Knowledge gaps* — prerequisite concepts missing for full understanding.
