#+title: org-sm
#+author: Lewis Liu

Minimal incremental reading for org-mode, inspired by SuperMemo.

* Card Types

*topic* — repeated reading, no grading required.
Interval grows by A-Factor each review.  org priority controls the rate:

| Priority    | A-Factor | Behavior                          |
|-------------+----------+-----------------------------------|
| =[#A]=      |      1.2 | slow growth, review frequently    |
| =[#B]=      |      1.5 | moderate growth                   |
| =[#C]= / none |    1.8 | fast growth, fades out quickly    |

*cloze* — fill-in-the-blank testing with ~{{answer}}~ markers, scheduled by [[https://github.com/bohonghuang/fsrs.el][FSRS]].
Answers are hidden as ~[___]~ during review; rated again / hard / good / easy.

* Installation

Requires Emacs 29.1+, org 9.6+, and the [[https://github.com/bohonghuang/fsrs.el][fsrs]] package.

#+begin_src elisp
(use-package org-sm
  :commands (org-sm-review-start org-sm-item-mark org-sm-item-extract org-sm-review-list)
  :hook (org-mode . org-sm-mode)
  :config
  ;; Set the list of org files to scan for SRS items
  (setq org-sm--files (directory-files-recursively "~/notes/" "\\.org$"))

  ;; Max length of auto-generated heading titles (default 40)
  (setq org-sm-title-max-length 50))
#+end_src

~org-sm-mode~ enables ~{{cloze}}~ font-lock only — safe to add to ~org-mode-hook~
without loading anything eagerly.

Suggested keybindings:

#+begin_src elisp
(with-eval-after-load 'org-sm
  (define-key org-mode-map (kbd "C-c s m")   #'org-sm-item-mark)
  (define-key org-mode-map (kbd "C-c s x")   #'org-sm-item-extract)
  (define-key org-mode-map (kbd "C-c s r")   #'org-sm-review-start)
  (define-key org-mode-map (kbd "C-c s SPC") #'org-sm-review-confirm)
  (define-key org-mode-map (kbd "C-c s d")   #'org-sm-item-dismiss)
  (define-key org-mode-map (kbd "C-c s q")   #'org-sm-review-abort)
  (define-key org-mode-map (kbd "C-c s l")   #'org-sm-review-list))
#+end_src

* Commands

| Command                 | Description                                     |
|-------------------------+-------------------------------------------------|
| ~org-sm-item-mark~       | Mark current heading as topic or cloze          |
| ~org-sm-item-extract~    | Extract selected region as child topic or cloze |
| ~org-sm-item-dismiss~    | Remove current item from SRS                    |
| ~org-sm-review-start~    | Start review session                            |
| ~org-sm-review-confirm~  | Confirm topic read / advance cloze state        |
| ~org-sm-review-abort~    | Abort current review session                    |
| ~org-sm-review-list~     | Browse all SRS items                            |

* Review Flow

*Topic:*
1. ~org-sm-review-start~ — opens due item, narrowed to its heading body.
2. Read the content.
3. ~org-sm-review-confirm~ — reschedules and moves to next item.

*Cloze:*
1. ~org-sm-review-start~ — opens due item with answers hidden as ~[___]~.
2. ~org-sm-review-confirm~ — reveals answers as ~[answer]~.
3. ~org-sm-review-confirm~ — rate: again / hard / good / easy.

Interval is always calculated from the originally scheduled date, so
early or late reviews do not distort future spacing.

* Extract

Select any text and call ~org-sm-item-extract~:

- *topic* — selected text becomes the child heading's body.
- *cloze* — full parent body is copied to the child; selection is wrapped as ~{{answer}}~.

The selection in the parent is replaced with an ~[[id:...][title]]~ link.

* Properties

| Property         | Type              | Description                     |
|------------------+-------------------+---------------------------------|
| ~SRS_TYPE~       | ~topic~ / ~cloze~ | Card type                       |
| ~SRS_LAST~       | ISO 8601          | Last review timestamp           |
| ~SRS_STATE~      | symbol            | FSRS state (cloze only)         |
| ~SRS_STABILITY~  | float             | FSRS stability (cloze only)     |
| ~SRS_DIFFICULTY~ | float             | FSRS difficulty (cloze only)    |
| ~SRS_STEP~       | int               | FSRS learning step (cloze only) |

~SCHEDULED~ is the due date for both types, used directly by ~org-sm-review-start~.
